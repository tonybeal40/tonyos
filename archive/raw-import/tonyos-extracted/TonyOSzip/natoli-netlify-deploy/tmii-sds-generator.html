1. Purpose

The purpose of this Software Design Specification (SDS) is to define the functional, architectural, and operational design of the Tool Management II (TM-II) software system. This document establishes the design baseline supporting Installation Qualification (IQ), Operational Qualification (OQ), and validated use in accordance with cGMP requirements.

2. Scope

This SDS applies to the Tool Management II software program and all associated modules. The TM-II application was written and configured using Magic Software XPA which is an off-the-shelf program with a large installed base.

Reference: SPEC-0169357 v1.0 - Natoli Tool System TM-II Software User Requirements Specification

3. System Description

Tool Management II (TM-II) is designed for tablet manufacturers. The program provides tablet manufacturers with tool control and inspection features to improve product quality and consistency, eliminate waste, and help ensure compliance with cGMP record keeping requirements.

3.1 Functional Overview

TM-II helps tablet manufacturers effectively manage their tool sets including:

  • Tool storage and inventory management
  • Inspection and tolerance reporting
  • Usage monitoring and history tracking
  • Tool location identification
  • Tool drawing file maintenance
  • Comprehensive reporting capabilities
  • Purchase order creation for replacement tools

4.2 Sets Directory

The Set Directory contains an entry for each set. Sets can be inventoried to produce Products.

FieldDescription
SET ID20 character alphanumeric unique ID or NDC #
SET NAME30 character alphanumeric name/description
NOTES150 character alphanumeric notes field

Sets Functions

  • Assemble - Create new sets from tools
  • Limits - View/edit usage history limits
  • Delete - Remove sets with confirmation
  • List - Print Set List Report
  • Stamp - Display Date/Time stamp
  • Current Status - Show usage status as percentage

Sets Reports

  • Summary Report - High-level activity overview
  • Usage Status Report - Usage as percentage of limits
  • Usage Report - Detailed usage history
  • Location Report - Tool location tracking
  • Maintenance Report - Maintenance history
  • Quarantine Report - Quarantined tools list
  • Discard Report - Discarded tools list
  • Check In/Out Report - Set movement history
  • Matching Report - Inspection analysis

7. Data Integrity and Audit Trail

The system records user ID, date, and time for all applicable actions per 21 CFR Part 11 requirements. The Electronic Audit function provides complete audit trail visibility.

  • All data modifications are tracked
  • User ID recorded for all changes
  • Date and time stamps for all actions
  • View Modify Stamps function available
  • Audit records are immutable
