1. Purpose and Scope

This Software Design Specification (SDS) defines the functional, architectural, and operational design of the Tool Management II (TM-II) software system. This document establishes the design baseline and demonstrates how each User Requirement from SPEC-0169357 v1.0 is addressed by the TM-II software design.

The scope of this SDS covers the Natoli Tooling Management TM-II software for use in the Process Area 5 compression area at the Newark, Delaware site. The associated Natoli Laser Vision System (LVS) interface is included; however, LVS has been separately validated.

2. System Description

Tool Management II (TM-II) is a comprehensive tool management software designed for tablet manufacturers. The system provides tool control and inspection features that improve product quality and consistency, eliminate waste, and ensure compliance with cGMP record keeping requirements.

2.1 Core Capabilities

  • Tool storage and inventory management
  • Inspection and tolerance reporting
  • Usage monitoring and history tracking
  • Tool location identification
  • Tool drawing file maintenance
  • Comprehensive reporting capabilities
  • Purchase order creation for replacement tools

2.2 System Architecture

TM-II is built on Magic Software XPA, an off-the-shelf development platform with a large installed base. The system utilizes three primary data structures:

  • Tool Directory - Main table containing entries for all punches and dies
  • Set Directory - Links tools into logical sets for production
  • Product Directory - Associates tool sets with products/tablets

3. Design Response to Software Requirements (URS Section 5)

3.1 Tool Directory Management

URS Reference Requirement Impact Design Response
5.1.1 The system shall be able to manage a tool directory that contains entries for punches and dies used in production. Direct TM-II provides a comprehensive Tool Directory as the main table. Each tool entry is assigned a unique system-generated ENTRY # and supports PART types: U (Upper Punch), L (Lower Punch), D (Die).
5.1.1.1 The tool directory shall contain: Identifier information, Supplier information, Tooling Material information, Tool Drawings, Tool Tolerance information, Overall Tool status information. Direct The Tool Directory includes all required fields:
  • Identifier: ENTRY #, HOB/DIE #, MACHINE ID, TYPE
  • Supplier: Supplier ID, P.O. Number fields
  • Material: Steel type, Max tip force fields
  • Drawings: Tool drawing file attachment support
  • Tolerance: SIZE, SHAPE, CUP DEPTH, LAND, QTY, TIPS
  • Status: INSPECTION date, LOCATION, MAINTENANCE, QUARANTINE status
5.1.1.2 The system shall have the ability to add, edit, or remove new tools to the tool directory. Direct TM-II provides Add, Edit, and Delete functions for the Tool Directory. The ADD/COPY NEW TOOLS user right controls this access. All modifications are audit-logged with user ID and timestamp.
5.1.1.3 The system shall permit tool data to be imported directly into the Tooling Directory. Non-Direct TM-II supports data import functionality through the DOWNLOAD DATA DISK utility, allowing bulk tool data import.
5.1.1.4 The system shall have the capability to search the tool directory list by text query. Non-Direct The Tool Directory provides search/filter functionality. Users can query by any field including ENTRY #, PART type, MACHINE ID, HOB/DIE #, TYPE, and status fields.

3.2 Set Directory Management

URS Reference Requirement Impact Design Response
5.1.2 The system shall be able to manage a set directory which contains an entry of all tools contained in each set. Direct TM-II provides a Set Directory with fields: SET ID (20 char), SET NAME (30 char), and NOTES (150 char). Each set entry links to associated tools.
5.1.2.1 The system shall have the ability to add, edit, or remove Sets. Direct Set management is controlled via the SETS and ASSEMBLE/DELETE SETS user rights. Functions include: Assemble (create), Limits (edit), Delete (remove), List, and Stamp.
5.1.3 The system shall be able to link Tools from the Tool Directory to Tool Sets in the Tool Sets Directory. Direct The Assemble function allows linking tools to sets. A tool can be in more than one set. The system maintains referential integrity between directories.

3.3 Product/Tablet Directory Management

URS Reference Requirement Impact Design Response
5.1.4 The system shall be able to manage a Product/Tablet Directory that contains an entry for every product or tablet. Direct TM-II provides a Products Directory for managing product/tablet entries. Access is controlled via ADD/CHANGE PRODUCTS user right.
5.1.4.1 The Product directory shall contain: Product overview information, Recommended default limits, Product tolerances as related to tool wear. Direct Product Directory includes:
  • Product Overview: Product identification and details
  • Recommended Limits: Default tablet limits per punch
  • Product Tolerances: Tolerances related to tool wear
  • Estimated Limits: Default values when inventoried
5.1.5 The system shall be able to link Tool Sets from the Set Directory to Products in the Product/Tablet Directory. Non-Direct Sets can be inventoried to Products. The Product Directory provides the link between Sets and the products they produce.

3.4 LVS Interface and Inspection

URS Reference Requirement Impact Design Response
5.1.6 The system must be able to interface with the Natoli Laser Vision System. Direct TM-II provides direct LVS integration. The USE LVS field enables automatic inspection with the Natoli LVS. Inspection data transfers seamlessly between systems.
5.1.12 The system shall have the ability to perform tool inspections in conjunction with a validated tooling measurement system. Direct Inspection module supports:
  • In-Process Inspection: Standard gages or Natoli LVS
  • Supplier Inspection: Incoming tool verification
Fields include: INSPECTION DATE, INSPECTOR ID, UNIT OF MEASURE, O.L. REFERENCE PT., up to 5 optional measurements.
5.1.13 The system shall be able to display results from completed tool inspections. Direct Inspection results are stored and displayed within the Tool Directory. The Matching Report provides inspection analysis. Historical inspection data is maintained.

3.5 Usage History and Reporting

URS Reference Requirement Impact Design Response
5.1.7 The system shall have the ability to maintain a history of tool usage which includes at a minimum the total number of tablets produced per tool. Non-Direct TM-II tracks usage history via UPDATE TOOL USAGE HISTORY function. Usage Status shows usage as percentage of limits. Limits function allows viewing/editing history limits.
5.1.8 The system shall be able to generate a maintenance report for each Tool. Direct Maintenance Report available through Set Reports. Tracks all maintenance activities with timestamps.
5.1.9 The system shall be able to generate a usage report for each tool. Direct Usage Report provides detailed usage history. Usage Status Report shows usage as percentage of limits.
5.1.10 The system shall be able to generate a Tool Summary report which includes at a minimum a high level overview of the activity for each Tool or Tool Set. Direct Summary Report provides high-level activity overview. Additional reports: Location Report, Check In/Out Report, Discard Report.
5.1.11 The system shall have the ability to quarantine Tools and Tool Sets as needed. Non-Direct UPDATE TOOL QUARANTINE function allows quarantine management. QUARANTINE field tracks status and date. Quarantine Report lists all quarantined tools.

3.6 Tool Check In/Out

The Tool Check In/Out module provides controlled workflow management for tool movement between storage locations and production areas. This functionality ensures complete chain of custody tracking and supports cGMP documentation requirements.

3.6.1 Check Out Process

Function Description
CHECK OUT Removes tools from inventory for production use. Updates tool LOCATION field and records transaction with timestamp and user ID.
Set Check Out Checks out entire tool set as a unit. All tools in set are updated simultaneously. Set maintains linked status during production.
Destination Tracking Records destination location (press, production area, maintenance) for each check out. Supports user-defined location codes.
Usage Assignment Associates check out with Product/Batch for usage tracking. Links to usage history for tablet count accumulation.

3.6.2 Check In Process

Function Description
CHECK IN Returns tools to inventory after production. Updates LOCATION field and usage history. Records return timestamp and user ID.
Usage Update Prompts for tablet count produced during check out period. Updates TABLETS PRODUCED field and usage percentage.
Condition Assessment Optional inspection prompt on check in. Can flag tools for maintenance, polishing, or quarantine based on condition.
Location Verification Confirms return to designated storage location. Updates storage position (cabinet, drawer, slot) as applicable.

3.6.3 Check In/Out Fields

Field Type Description
TRANSACTION ID System Generated Unique identifier for each check in/out transaction
TRANSACTION TYPE Selection CHECK IN or CHECK OUT
ENTRY # Reference Tool Directory entry number
SET ID Reference Set Directory ID (if checking out as set)
FROM LOCATION Text (20 char) Origin location for the tool
TO LOCATION Text (20 char) Destination location for the tool
PRODUCT Reference Product being manufactured (for usage tracking)
BATCH/LOT Text (30 char) Batch or lot number for traceability
TABLETS PRODUCED Numeric Count entered on check in
USER ID Reference User performing the transaction
DATE/TIME Timestamp System-recorded date and time
NOTES Text (150 char) Optional comments or observations

3.6.4 Check In/Out User Rights

User Right Description
UPDATE TOOL LOCATION Required to perform check in/out operations
UPDATE TOOL USAGE HISTORY Required to update tablet counts on check in
VIEW CHECK IN/OUT HISTORY Required to view historical transactions

3.7 Utilities

The Utilities module provides system administration and maintenance functions for TM-II. These tools support data management, system configuration, and operational tasks.

3.7.1 Data Management Utilities

Utility Description
DOWNLOAD DATA DISK Imports tool data from external source (USB, network share). Supports bulk data loading for new tool inventory. Maps external data fields to TM-II directory structure.
BACKUP DATABASE Creates backup copy of entire TM-II database. Supports scheduled and on-demand backup. Recommended before system updates or major changes.
RESTORE DATABASE Restores database from previous backup. Requires Supervisor-level access. All current data replaced with backup contents.
EXPORT DATA Exports tool, set, or product data to external file formats. Supports CSV, Excel, and text formats. Configurable field selection.
ARCHIVE DATA Moves inactive/obsolete records to archive tables. Reduces active database size while preserving historical records. Archived data remains searchable.

3.7.2 System Configuration Utilities

Utility Description
MODIFY SETUP TABLES Configures system lookup tables: Machine Types, Locations, Suppliers, Steel Types, Tool Types. User right controls access.
SYSTEM PREFERENCES Configures global system settings: default units of measure, date format, report headers, company information.
USER MANAGEMENT Add, modify, or remove user accounts. Assign user rights and access levels. Reset passwords. View user activity logs.
LICENSE MANAGEMENT View HASP license key information. Display licensed features and user count. Verify license validity.
LVS CONFIGURATION Configure Laser Vision System connection settings. Set communication parameters. Test LVS connectivity.

3.7.3 Maintenance Utilities

Utility Description
DATABASE MAINTENANCE Optimize database performance. Rebuild indexes. Reclaim storage space. Recommended for periodic maintenance.
AUDIT LOG VIEWER View complete audit trail of system activity. Filter by date range, user, or action type. Export audit records.
ERROR LOG VIEWER View system error and warning messages. Useful for troubleshooting. Includes timestamp and error details.
PURGE OLD RECORDS Removes records older than specified retention period. Requires Supervisor access. Generates purge report before execution.

3.7.4 Reporting Utilities

Utility Description
REPORT DESIGNER Create custom reports using available data fields. Save report templates for reuse. Configure output format and layout.
SCHEDULED REPORTS Configure automatic report generation. Set schedule (daily, weekly, monthly). Email or save to network location.
PRINT MANAGER Configure default printers for reports and labels. Set print preferences. Manage print queue.

3.7.5 Utility Access Rights

User Right Controlled Functions
MODIFY SETUP TABLES Access to all setup table configuration
ELECTRONIC AUDIT View audit trail and system logs
BACKUP/RESTORE Database backup and restore operations
EXPORT DATA Export tool and set data to external files
USER ADMINISTRATION Manage user accounts and permissions
SYSTEM CONFIGURATION Modify system preferences and settings

3.7.6 Main Utilities Menu Functions

The following utilities are accessible from the main UTILITIES menu on the TM-II home screen:

Utility Description
PURCHASE ORDERS Creates and manages purchase orders for replacement tooling. Generates POs based on tool wear, usage limits, or manual request. Links to supplier directory for vendor information. Tracks PO status from creation through receipt. Supports export to ERP/procurement systems.
MACHINE SETUP Configures tablet press machine definitions within TM-II. Defines machine ID, type (rotary, single station), station count, and location. Associates machines with tool sets for production tracking. Manages machine-specific inspection parameters and tolerance defaults.
MACHINE SETUP PROCEDURES Stores and retrieves standard operating procedures for machine setup activities. Links SOPs to specific machine types and tool configurations. Provides step-by-step guidance for operators during tool installation. Supports attachment of reference documents and images.
SETUP TABLES Maintains system lookup tables and reference data. Includes: Supplier Master List, Steel Type Codes, Tool Type Classifications, Location Codes, Machine Type Definitions, Unit of Measure settings, and custom field definitions. Changes are audit-logged per 21 CFR Part 11 requirements.
CHANGE TOOL IDs Modifies tool identification numbers in controlled manner. Supports bulk ID changes for tool sets. Maintains audit trail of all ID modifications with before/after values. Requires elevated user rights for execution. Updates all related records (sets, products, history) automatically.
S.O.P. MAINTENANCE Standard Operating Procedure document management within TM-II. Create, edit, and version-control SOPs for tool handling, inspection, and maintenance activities. Links SOPs to specific operations and tool types. Tracks SOP revision history and approval status. Supports electronic signatures for SOP approval per 21 CFR Part 11.
ELECTRONIC AUDIT Complete audit trail viewer for all system transactions. Displays user ID, timestamp, action type, and before/after values for all data modifications. Filter by date range, user, table, or action type. Export audit records to CSV or PDF. Immutable records ensure data integrity for regulatory inspections. Required for 21 CFR Part 11 compliance.

4. Design Response to Hardware Requirements (URS Section 6)

URS Reference Requirement Impact Design Response
6.1 The system shall utilize a Windows-based computer running Windows 10 or newer. Non-Direct TM-II is designed for Windows operating systems. Fully compatible with Windows 10 and Windows 11. Built on Magic Software XPA platform.
6.2 The system shall have a minimum of 64MB memory. Non-Direct TM-II requires minimum 64MB RAM. Recommended: 4GB or higher for optimal performance.
6.3 The system shall have a minimum of 800MB available disk space. Non-Direct Installation requires minimum 800MB disk space. Recommended: 2GB+ for database growth.
6.4 The system shall have a DVD drive and a USB port. Non-Direct DVD drive supported for installation media. USB port required for HASP license key and data transfer.

5. Design Response to Utility Requirements (URS Section 7)

TM-II is designed as a standalone software application. No external utility connections (power, water, compressed air, HVAC) are required beyond standard computer power requirements.

6. Design Response to Computerized System Compliance (URS Section 8)

6.1 21 CFR Part 11 Compliance

URS Reference Requirement Impact Design Response
8.1 The system shall maintain an electronic audit trail that records user ID, date, and time for all applicable actions. Direct Electronic Audit function provides complete audit trail. All modifications logged with User ID, Date, Time. View Modify Stamps shows change history. Audit records are immutable.
8.2 The system shall provide user authentication with unique user identification. Direct User authentication requires unique User ID and password. Default: Supervisor (not case-sensitive). Security module manages user accounts and access rights.
8.3 The system shall provide role-based access control. Direct 25+ configurable user rights including: MODIFY INSPECTIONS, CREATE INSPECTIONS, UPDATE TOOL LOCATION, UPDATE TOOL USAGE HISTORY, UPDATE TOOL QUARANTINE, ADD/COPY NEW TOOLS, ASSEMBLE/DELETE SETS, MODIFY SETUP TABLES, ELECTRONIC AUDIT, and more.

6.2 License and Security

System authorization is enforced through a HASP hardware license key. The USB-based dongle must be present for application execution. This provides physical security control and license enforcement.
