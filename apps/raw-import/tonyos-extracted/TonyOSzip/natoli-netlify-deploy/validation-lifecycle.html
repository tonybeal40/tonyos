SDS Software Design Specification

Tool Management II (TM-II)
Software Design Specification

1. Purpose

The purpose of this Software Design Specification (SDS) is to define the functional, architectural, and operational design of the Tool Management II (TM-II) software system. This document establishes the design baseline supporting Installation Qualification (IQ), Operational Qualification (OQ), and validated use in accordance with cGMP requirements.

2. Scope

This SDS applies to the Tool Management II software program and all associated modules. The TM-II application was written and configured using Magic Software XPA which is an off-the-shelf program with a large installed base. No structural testing is therefore included in this protocol.

3. System Description

Tool Management II (TM-II) is designed for tablet manufacturers. The program provides tablet manufacturers with tool control and inspection features to improve product quality and consistency, eliminate waste, and also help ensure compliance with cGMP record keeping requirements.

3.1 Tool Directory

The Tool Directory table contains entries for the punches and dies in your organization. Each entry contains information about the tooling including quantity, type, machine type, steel, etc.

ENTRY # - System generated unique number assigned to each tool
PART - Tool Part type: U (Upper Punch), L (Lower Punch), D (Die), etc.
HOB/DIE # - Number associated with the particular Hob or Die
MACHINE ID - Machine for which the tool is designed
TYPE - Tool type: B type, D type, etc.
QTY - Quantity of tools available
TIPS - Number of tips on a particular tool
SIZE - Size of the tablet the tool produces
SHAPE - Shape of the tablet the tool produces
CUP DEPTH - Size of the punch cup depth
LAND - Size of the punch land area
3.2 Sets Directory

The Set Directory contains an entry for each set. Sets can contain multiple tools and can be inventoried to produce Products.

SET ID - 20 character alphanumeric unique ID or NDC #
SET NAME - 30 character alphanumeric name/description
NOTES - 150 character alphanumeric notes
3.3 Products Directory

The Products table contains an entry for every Product or Tablet with 'tablet limits per punch' default amounts.

4. Security Design

System access is controlled through user authentication and role-based authorization. A HASP hardware license key enforces application execution.

Default User ID - Supervisor (not case sensitive)
User Rights - SETS, AUDIT, MODIFY INSPECTIONS, CREATE INSPECTIONS, UPDATE TOOL LOCATION, etc.
5. Audit Trail and Data Integrity

The system records user ID, date, and time for all applicable actions per 21 CFR Part 11 requirements.
